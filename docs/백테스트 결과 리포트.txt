 국면 전환 손실의 근본 원인

현재 국면 판단은 매 캔들마다 즉시 전환됩니다:

 - ADX가 24 → 25로 1포인트만 올라도 횡보→추세 전환
 - 전환 직후 기존 포지션의 청산 규칙이 바뀌면서 타이밍 불일치 발생

대표적 손실 패턴 3가지

패턴 A: 횡보 포지션이 추세에 역행

횡보 Short 보유 중 → 가격 상승 → ADX 상승 → 추세(UP) 전환
→ 추세 Long 신호 발생 → 기존 Short "반대 포지션 전환" 으로 손실 청산
→ 바로 50x Long 진입 → 추세가 진짜가 아니면 또 손실

패턴 B: 추세 진입 직후 국면 복귀

추세(DOWN) 판단 → 50x Short 3회 물타기 진입
→ 가격 반등 → ADX 하락 → 횡보로 복귀
→ 50x 포지션이 횡보 BB 청산 규칙(BB% 65%)에 걸려 손실 청산

패턴 C: 강제청산 (-150 USDT)

추세 50x 진입 → 국면 전환 → 청산 시그널 오기 전에 마진 소진
→ 50x라서 2% 역행만으로도 마진 전액 소진

해결 방안 (수익 유지, 손실 감소 중심)

방안 1: ADX 상승 감지 시 횡보 진입 억제 (효과 최대)

현재는 ADX < 25이면 무조건 횡보로 판단하지만, 
ADX가 20 이상이면서 상승 중이면 추세 전환 직전일 가능성이 높습니다.

  현재: ADX=22 (횡보) → 반전매매 진입 허용
  개선: ADX=22 + ADX 상승중 → 신규 진입 금지 (관망)

 - 기존 포지션의 청산은 그대로 유지 (수익에 영향 없음)
 - 신규 진입만 막으므로 추세 전환 직전의 역방향 포지션을 원천 차단
 - 수익 감소 거의 없음: 횡보장 PF가 0.76이므로 이 구간의 진입을 줄이면 오히려 이득

방안 2: 횡보 물타기 시 ADX 조건 추가

물타기는 손실을 키우는 핵심입니다. 2차/3차 물타기에 ADX 필터를 추가합니다.

  현재: BB% 도달하면 무조건 물타기
  개선: ADX > 18이면 2차 물타기 금지, ADX > 15이면 3차 금지

 - 1차 진입만 하고 물타기 없이 손절되면 마진 50 USDT 손실
 - 3차까지 물타기 후 손절되면 마진 150 USDT 손실
 - 물타기 억제만으로 최대 손실이 1/3로 감소

방안 3: 국면 전환 쿨다운 (cooldown)

국면이 바뀐 직후 N캔들(예: 3~5캔들) 동안 신규 진입을 금지합니다.

  [횡보 → 추세 전환 감지]
    → 3캔들 관망 (추세가 진짜인지 확인)
    → 확인 후 추세추종 진입

 - 추세 시작 초기의 가짜 브레이크아웃 손실을 방지
 - 진짜 추세는 3캔들 후에도 유효하므로 대형 수익 거래는 대부분 포착 가능
 - 수익 감소: 소폭 (진입 가격이 약간 불리해짐), 손실 감소: 상당 (가짜 전환 필터링)

방안 4: 추세장 트레일링 스톱

현재 추세 익절은 고정 +3%입니다. 트레일링 스톱을 도입하면:

  현재: PnL +3% 도달 → 즉시 익절
  개선: PnL +2% 도달 → 트레일링 시작, 고점 대비 -1% 하락 시 청산

 - 추세가 강하면 +3% 이상 수익 가능 (수익 증가)
 - 추세가 약해지면 수익을 지키며 조기 청산 (손실 감소)
 - Top 거래 #44(+524.67)처럼 대형 수익은 더 커질 수 있음

방안 5: 국면 확신도 기반 포지션 사이징

이진 판단(횡보/추세) 대신 연속 점수로 포지션 크기를 조절합니다.

  현재: strength >= 2.0 → 추세 (풀사이즈), < 2.0 → 횡보 (풀사이즈)
  개선:
    strength 0~1.0  → 횡보 풀사이즈 (100%)
    strength 1.0~2.0 → 횡보 축소 사이즈 (50%)  ← 전환 구간
    strength 2.0~3.0 → 추세 축소 사이즈 (50%)  ← 전환 구간
    strength 3.0+    → 추세 풀사이즈 (100%)

 - 확신도가 낮은 전환 구간에서 자동으로 마진을 줄임
 - 잘못된 국면 판단의 손실이 반으로 줄어듦
 - 확실한 구간에서는 기존과 동일한 수익

백테스트 한 결과가 report_1h_165414.txt, report_1h_165701.txt인데, 
다른 조건 다 똑같이 하고, capital만 100 -> 1000으로 변경했는데 수익률이 이렇게 많이 차이날까?

Answer:

원인은 max_margin_per_entry = 50 USDT 고정 상한 때문입니다.

마진 계산 로직을 보면 (engine.py:175-184):

  margin = self.capital * signal.position_ratio   # 자본의 N%
  max_margin = self.max_margin_per_entry           # 고정 50 USDT
  margin = min(margin, self.capital, max_margin)

Capital 100일 때:

 - 초기 마진 = 100 × 5% = 5 USDT (50 상한에 안 걸림)
 - 자본이 200으로 불어나면 → 마진도 10 USDT로 증가 → 복리 효과 발생
 - 자본이 1000 넘어야 비로소 50 USDT 상한에 걸림

Capital 1000일 때:

 - 초기 마진 = 1000 × 5% = 50 USDT → 처음부터 상한에 걸림
 - 자본이 1500이 되어도 마진은 여전히 50 USDT → 복리 효과 없음, 자본 대비 투입 비율이 계속 줄어듦

리포트 비교를 보면 이게 명확합니다:

월	  Capital 100	  Capital 1000
1월	  +103.86%	    +58.70%
2월	  +44.55%	      +39.92%
3월	  +26.59%	      +26.59%

1월에 격차가 가장 크고, 3월에 동일해지는 이유
 — capital 100도 3월쯤이면 자본이 충분히 커져서 50 USDT 상한에 걸리기 때문입니다.

해결 방법: margin_pct 옵션을 사용하면 됩니다. 
        고정 USDT 대신 자본 대비 비율로 마진을 설정하면 자본 규모에 관계없이 동일한 수익률이 나옵니다.

  # config.yaml에서
  backtest:
    margin_pct: 0.05   # 자본의 5%를 1회 마진으로 (max_margin_per_entry 대신)

이렇게 하면 capital 100이든 1000이든 동일한 수익률이 나옵니다.

1. 추세 진입 조건 강화

현재 ADX 20 이상이면 추세로 판단하는데, 이것이 너무 낮다.

  # config.yaml
  strategy:
    adx_entry_block: 25.0    # 20 → 25 (더 확실한 추세에서만 진입)
    adx_rise_lookback: 5     # 3 → 5 (ADX 상승 확인 기간 늘리기)

포지션 #1 (ADX=21.6), #4 (ADX=26.1이지만 BBW=0.70)은 이 필터로 걸러진다.

2. BBW 최소 기준 추가 검토

#4처럼 BBW=0.70 (밴드 매우 좁음)에서 추세추종 진입하면 위험하다. 
밴드가 좁다는 건 변동성이 낮다는 뜻인데 50x로 들어가면 조금만 역행해도 큰 손실이다. 
BBW가 일정 수준 이상일 때만 추세 진입하는 필터가 있으면 좋다.

3. BB MTF 전략 사용

현재 strategy.name: "bb" (단일 타임프레임)이다. 
"bb_mtf"로 바꾸면 상위/하위 타임프레임 국면을 교차 확인해서 허위 추세 전환을 필터링할 수 있다.

  strategy:
    name: "bb_mtf"    # "bb" → "bb_mtf"

#1의 경우 1h에서 trend_up이지만 4h나 30m에서 sideways라면 진입을 차단할 수 있다.

4. 추세 레버리지 낮추기

50x는 2% 역행이면 마진 전액 손실이다 (#14가 정확히 이 케이스).

  backtest:
    leverage_max: 30      # 50 → 30
    leverage_min: 20      # 50 → 20

수익도 줄지만, #1(-9.27), #14(-4.98) 같은 대형 손실이 줄어들어 전체적으로 유리하다.

5. 물타기 간격 확대 (코드 수정 필요)

현재 매 캔들(1h)마다 물타기가 가능해서 3시간 만에 3회 풀 진입된다. 
최소 3~5캔들 간격을 두면 가격 확인 후 진입하므로 허위 추세에 물타기하는 위험이 줄어든다.

6. 횡보 Short 진입 조건 강화

횡보 Short 3전 3패. BB% 100%보다 BB% 110~120% 이상에서만 Short 진입하거나, 
ADX가 하락 중일 때만 진입하는 조건을 추가하면 추세 전환 직전의 역방향 진입을 피할 수 있다.

--------------------------

즉시 적용 가능한 config 변경

  strategy:
    name: "bb_mtf"              # 다중 TF로 허위 추세 필터링
    adx_entry_block: 25.0       # 추세 진입 ADX 기준 상향

  backtest:
    leverage_max: 30             # 최대 레버리지 하향
    leverage_min: 20             # 최소 레버리지 하향
    sideways_leverage_max: 10    # 횡보장 레버리지도 하향

이 설정으로 백테스트를 다시 돌려보면 #1(-9.27), #14(-4.98) 같은 대형 손실이 
줄어들면서 전체 수익률이 개선될 가능성이 높습니다. 
다만 13일은 짧은 기간이므로, 이전에 했던 1~3월 데이터 백테스트와 함께 비교해보는 게 좋습니다.

Q1. 실거래에서 --capital 100 으로 실행했을 경우, 실제 계좌(예수금=free) 금액이 100이 넘을 때 
기준에서 넘는 금액을 수익으로 계산하는데 그건 total capital로 설정하고, 
capital은 실금액이 어떻든 프로그램 시작 금액은 100으로 해서 이후 그 100에서 수익이 어떻게 되는지 판단하고, 
마진 5%도 그 100에서 150이 됐으면 150의 5%로 계산하여 실제 총 금액은 무시하는 방법을 추가해줘
(기존 총합으로 하는 건 total capital 방식으로 그대로 사용가능)

A:
가상자본(virtual capital) 모드 추가

개념:

모드	          동작
total (기본)	  거래소 실잔고 = capital. 기존 동작 그대로
virtual	       --capital 100이면 100에서 시작, PnL만 반영하여 가상 자본 추적. 실잔고 무시

virtual 모드에서 달라지는 점:

 - _sync_balance() — 실잔고를 capital에 덮어쓰지 않음 (로깅만)
 - 마진 계산 — 가상 자본 기준 (100→150이면 150의 5%)
 - 상태 저장/복원 — virtual_capital 필드로 가상 자본 값 보존
 - 진입/청산 — 기존과 동일하게 self.capital에서 차감/복원 (virtual이든 total이든 코드 동일)

사용법:

  # config.yaml
  trader:
    capital_mode: "virtual"

  # CLI (config보다 우선)
  python -m src.main trade -e binance_futures -s btc --capital 100 --capital-mode virtual

수정 파일

 - config.py — TraderConfig에 capital_mode 필드 추가
 - live_trader.py — _sync_balance() virtual 분기, 상태 저장/복원, 헤더 표시
 - main.py — --capital-mode CLI 옵션, trader_kwargs 전달
 - config.yaml — capital_mode 설정 및 설명 추가
 - CLAUDE.md, CHANGELOG.md — 문서 업데이트
