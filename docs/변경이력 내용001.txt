A:
제안하신 방향은 현재 V6의 문제점(후행지표 의존 + squeeze 강제 sideways)을 근본적으로 해결하는 방향입니다. 
차트 직관("폭이 커지고 이전 레벨을 뚫으면 추세")을 직접 코드화하는 접근이라, 
앞서 진단한 04-17 지연과 04-18~20 하락 오판 모두 완화될 가능성이 높습니다.

다만 구현 전에 정의를 명확히 해야 할 지점이 몇 개 있어서 확인이 필요합니다.

사용자 안 요약 (재정리)

국면	    조건 A (BB 폭)	                조건 B (가격 돌파)
추세 상승	현재 폭 > 최근 N봉 평균 폭	    이전 저항선 상향 돌파
추세 하락	현재 폭 > 최근 N봉 평균 폭	    이전 지지선 하향 돌파
횡보	    현재 폭 ≈ 최근 N봉 평균 폭	    돌파 없음

두 조건을 AND로 묶는 게 자연스럽습니다 (폭은 커졌는데 돌파 없음 = 단순 변동성 증가 ≠ 추세).

보완이 필요한 4가지 정의

1) "이전 저항선/지지선"의 기준

두 해석이 가능합니다:

 - (a) BB 상/하단 레벨: max(bb_upper[t-N:t-1]), min(bb_lower[t-N:t-1])
 - (b) 가격 고점/저점: max(close[t-N:t-1]), min(close[t-N:t-1])

→ 권장: (b) 종가 기준. (a)는 BB 자체가 가격 따라 움직여서 "가격이 BB 상단을 뚫음"이 거의 항상 만족되어 
돌파 감지력이 약합니다. 차트의 "저항/지지선"은 실제 가격 히스토리에 형성되므로 (b)가 직관과 일치합니다.

2) "별 차이 없음" vs "커짐"의 임계

단순 > 는 노이즈에 취약 → 버퍼 필요:

 - 확장: width_now > width_avg_N * 1.05 (5% 초과)
 - 유사: 0.95 * avg ≤ width_now ≤ 1.05 * avg

3) 돌파 버퍼 (wick 노이즈 방지)

 - close > prev_resistance * (1 + 0.001) (0.1% 여유)
 - 잠깐 찔렀다 돌아오는 꼬리 캔들이 trend 트리거 되는 걸 막음

4) 추세 종료 조건 (사용자 정의 누락)

제안은 "진입" 조건만 있어서 언제 추세 끝났다고 볼지가 미정의입니다. 선택지:

 - (α) 매 캔들 재판정 (순수): 해당 캔들이 돌파+확장 조건 불만족 → 즉시 sideways
 - (β) Hysteresis (V5 방식): trend 감지 후, sideways 조건을 K캔들(기본 3) 연속 만족해야 전환
 - (γ) 반대편 돌파까지 유지: trend_up은 하단 돌파 전까지 유지

→ 권장: (β) Hysteresis. (α)는 너무 빈번히 튀고, (γ)는 V6의 sticky 문제 재현 위험.

최종 수식 (권장)

N = width_lookback  # 기본 5
width = bb_upper - bb_lower
width_avg = width.rolling(N).mean().shift(1)  # 현재 캔들 제외 평균
width_expanded = width > width_avg * 1.05

# 가격 레벨 (종가 기준)
prev_high = close.rolling(N).max().shift(1)
prev_low  = close.rolling(N).min().shift(1)
break_up   = close > prev_high * (1 + 0.001)
break_down = close < prev_low  * (1 - 0.001)

# 원시 국면
raw_trend_up   = width_expanded & break_up
raw_trend_down = width_expanded & break_down

# Hysteresis: sideways 전환은 K=3캔들 연속 양쪽 다 false 일 때

파라미터 제안 (BB V7 신설)

break_buffer_pct	0.001	    돌파 최소 여유 (0.1%)

3. 이 차트에 최적화된 국면 판단 설계 (5개 규칙 조합)

규칙 A — 캔들 몸통 누적 방향성 (1차 빠른 감지)

	recent_n = 5 (1h 기준 5시간)
	body_sign = sign(close - open)  # +1/-1/0
	score = Σ body_sign · (|close - open| / ATR14)

	score ≥ +2.0 → trend_up 후보
	score ≤ −2.0 → trend_down 후보
	else         → sideways 후보

  - Apr 2~3 하락처럼 큰 빨간 몸통이 연속되는 구간에서 즉시 trigger.

  - 후행 평균을 쓰지 않으므로 ADX보다 빠릅니다.

규칙 B — BB 외부 체류 캔들 수 (페이크 필터)

	out_up   = close > upper_bb 인 연속 캔들 수
	out_down = close < lower_bb 인 연속 캔들 수

	out_up ≥ 2 AND 규칙A = trend_up 후보   → trend_up 확정
	out_down ≥ 2 AND 규칙A = trend_down 후보 → trend_down 확정
	out_up/down = 1 AND 다음 캔들이 중단 회귀 → sideways로 강제

  - Apr 8의 1캔들 스파이크는 out_up=1 → sideways 유지 (현재 V6 squeeze 로직이 놓치는 부분).

규칙 C — 스윙 피벗 구조 (구조적 추세)

1시간봉에서 ZigZag 3% 이상의 pivot만 인정:

	최근 2개 pivot이 HH-HL 형성 → trend_up
	최근 2개 pivot이 LH-LL 형성 → trend_down
	그 외                       → sideways

이 차트 검증:

  - Apr 1 고점 85 → Apr 3 저점 76.5 = LL → trend_down 확정 (Apr 3 아침에 확정됨 — 하락 중~중반)

규칙 D — 중단(20MA) vs 종가 위치의 지속성

	위치 = sign(close - bb_mid)
	최근 N캔들 모두 같은 부호 AND |close - bb_mid| / bb_mid 가 단조 증가
	  → 강한 일방향 추세

  - Apr 2~3: 연속 캔들이 모두 mid 아래 + 괴리율 증가 → trend_down 보강.
  - Apr 9~13: 부호가 +/− 번갈아감 → sideways 보강.

V5/V7의 아이디어를 더 엄격하게:

	trend_up ↔ trend_down 전환은 sideways를 반드시 경유

  - Apr 8 스파이크 단일 캔들이 regime을 뒤집지 못하게 함.

Q: 2026-04-02 05:00 사유 추세장 반전매수 - 추세장에서 반전매수를 왜 하는지? 언제 추세가 바뀔줄 알고?
반전매매는 횡보장에서만 하는 거 아닌가?

A:
좋은 지적입니다. 코드를 정확히 읽어보니 설계 자체에 혼선이 있습니다.

"추세장 반전매수"가 찍히는 조건 (trend.py:200-238)

  # BB 하단(bbp ≤ 0.20) 도달 시:
  if not is_uptrend and trend_confirmed:    # 하락 추세 + 확인됨
      # → Short 추세추종
  else:                                     # 그 외
      # → Long 진입 ← 여기가 "추세장 반전매수"

즉 이 reason이 찍히는 케이스는 두 가지:

  1. regime = TREND_UP 인데 BB 하단 터치 → 상승 추세 중 일시 눌림 → Long 진입

    - 이건 엄밀히는 "눌림목 매수"(pullback buy) 이지 "반전"이 아닙니다. reason 문자열이 잘못 붙은 셈.
    - 투자 상식상 허용 가능한 전략.

  2. regime = TREND_DOWN 인데 trend_confirmed=False + BB 하단 → Long 진입

    - 이건 진짜 반전 시도. "하락 추세처럼 보이지만 MACD/RSI/Volume 중 2개가 미확인이니 바닥 잡아보자" 라는 로직.
    - 사용자가 지적한 "떨어지는 칼날 잡기" 그 자체. 가장 위험한 케이스.

    ※ 참고로 BB 상단 쪽 대칭 케이스(is_uptrend and trend_confirmed 외의 else에서 Short 진입)도 
    같은 문제가 있습니다 — reason은 "추세장 반전매도".

사용자 지적은 전적으로 타당

정석적으로:

  - 추세장 = 추세추종만 (trend-following)
  - 횡보장 = 반전매매 (mean-reversion)

현재 구조는 추세장 속에 반전매매를 섞어놓은 상태라, 규칙상 이익이 크게 날 수 있는 추세 구간에서도 
역방향 신호가 발생해서 슬리피지/손절 누적이 생깁니다. 이전에 보여주신 SOL/USDT 4월 차트에서 
Equity가 1000 부근에서 횡보한 이유 중 하나입니다.

2026-04-02 05:00 의 실제 상황

정확히 어느 케이스(1번/2번)였는지는 signal.log의 V9 전환 로그 + BB% 값을 보면 알 수 있습니다. 
원하시면 해당 시점 주변 로그를 파싱해서 확인해드릴 수 있습니다.

개선 방향 (선택)

Option A — 보수적 (추천): 추세장에서는 추세추종만 허용. else 브랜치 제거.

  - 상승 추세 + BB 하단 → 아무 것도 안 함 (다음 상단 돌파 대기)
  - 하락 추세 미확인 → sideways로 downgrade하여 반전매매 로직으로 이관

Option B — 선별적 허용: 케이스 1(눌림목)만 허용, 케이스 2(역추세 반전)는 차단.

  - 현재 reason을 "눌림목 매수" / "반대추세 반전"으로 구분해 로그에서 바로 보이게
  - 케이스 2는 완전 차단

Option C — 현상 유지 + 리네이밍만: 동작은 유지하고 reason 문자열만 정확하게 교체.

어떻게 하시겠습니까?

제안: Option A를 BB V9 전용으로 적용하는 게 가장 깔끔합니다. 
V9의 regime이 정확하게 추세를 잡는다는 가정 위에서 추세장은 한 방향 진입만 허용하는 방향. 
V1~V8은 하위 호환을 위해 그대로 두고, V9에서 _trend_signals만 오버라이드하면 됩니다.

적용할까요? 아니면 먼저 2026-04-02 05:00 주변 로그부터 확인해서 실제로 케이스 1인지 2인지 보고 싶으신지요?

Q: 2026-04-09 08:00 횡보 반전매수가 나옴

A:
근본 원인

  1. 실제로는 하락 추세가 진행 중이었지만 03:00 → 08:00의 "6시간 숨고르기"가 
      V9 hysteresis(3캔들 연속 sideways) 조건을 충족시켜 06:00에 sideways로 복귀.
  2. sideways 복귀 즉시 반전매매 로직이 활성화 → 08:00에 BB 하단 근처(bbp≈0.07)라서 Long 반전매수 체결.
  3. 바로 다음 캔들(09:00)에서 trend_down 재진입 → 하락 3시간 연속(09~13:00) → 물타기 또는 손절 후보.

즉 V9의 hysteresis 통과 직후 곧바로 반전매매 진입이 whipsaw의 원인입니다. 
이건 V4 전략이 이미 경험한 문제 패턴이고, V4는 cooldown_candles 로 이를 해결합니다.

V4 에서 이미 있는 해결책

CLAUDE.md의 V4 설명 그대로:

    BB V4 전략: BB V2 상속 + 국면 전환(trend→sideways) 쿨다운(cooldown_candles, 기본 5)
       — 전환 직후 N캔들 동안 횡보 신규진입만 차단 (물타기/청산 영향 없음)

V9는 V4와 독립적으로 분기됐는데, V9에 V4의 cooldown 로직을 얹으면 이번 케이스가 정확히 걸러집니다:

  - 06:00에 sideways 복귀 → 06~10:00 cooldown(5캔들) → 신규 횡보 진입 차단
  - 08:00에 반전매수 시도 → cooldown 걸려서 관망
  - 09:00에 trend_down 재진입 → 영향 없음 (물타기가 아니라 신규 진입이었으므로)

제안

V9에 cooldown_candles 파라미터 도입 (기본 3~5). V4 로직을 그대로 포팅:

  - detect_regime 이후 regime 시리즈에서 trend→sideways 전환 지점 찾아 _cooldown_until_ts 에 기록
  - generate_signals 의 sideways 분기에서 현재 ts가 cooldown 안이면 횡보 신규 진입만 차단
  - 물타기/청산은 영향 없음 → 기존 포지션 관리는 그대로

구현하시겠습니까? 기본값은 V4와 동일하게 cooldown_candles=5로 시작해서, 이번 차트(06:00 복귀 → 10:00 cooldown 종료) 기준으로 09:00 이후 진입 차단하는 효과를 얻을 수 있습니다. 더 빠른 복귀가 필요하면 3으로 낮출 수 있고요.
